# Инструкция по исправлению и деплою сайта на Vercel

## Что было исправлено

### 1. Проблемы с API на Vercel
- **Было**: API endpoints не работали на Vercel из-за неправильной конфигурации
- **Стало**: Созданы правильные serverless функции в папке `api/`
  - `fear.js` - для проверки банов
  - `player.js` - для получения данных игрока

### 2. Проблемы с конфигурацией Vercel
- **Было**: Файл `vercel.json` содержал неправильные маршруты
- **Стало**: Обновлен `vercel.json` с правильными rewrite правилами

### 3. Проблемы с путями в JavaScript
- **Было**: JavaScript файлы использовали неправильные пути к API
- **Стало**: Исправлены пути в `ConfigChecker.js` и `APIClient.js`

### 4. Проблемы с локальным сервером
- **Было**: Сервер не работал правильно локально
- **Стало**: Обновлен `server.js` с правильной обработкой CORS и прокси

## Как проверить работу локально

### Вариант 1: Через скрипты
```bash
# Windows (CMD)
start-local.bat

# Windows (PowerShell)
.\start-local.ps1

# Linux/Mac
chmod +x start-local.sh
./start-local.sh
```

### Вариант 2: Вручную
```bash
cd check-page-standalone
npm install
npm start
```

### Проверка работы
1. Откройте браузер и перейдите по адресу: http://localhost:3002
2. Загрузите тестовый файл `test-config.vdf`
3. Проверьте, что игроки правильно отображаются

## Как деплоить на Vercel

### Вариант 1: Через Vercel CLI
```bash
npm install -g vercel
cd check-page-standalone
vercel
```

### Вариант 2: Через веб-интерфейс
1. Перейдите на https://vercel.com
2. Нажмите "Add New" → "Project"
3. Импортируйте папку `check-page-standalone`
4. Настройки будут автоматически определены из `vercel.json`

### Вариант 3: Через Git
```bash
git init
git add .
git commit -m "Fixed Vercel deployment"
git remote add origin https://github.com/yourusername/fear-protection-check.git
git push -u origin main
# Затем импортируйте репозиторий в Vercel
```

## Решение проблем

### Если API не работает на Vercel
1. Проверьте логи в Vercel Dashboard → Functions
2. Убедитесь, что файлы в папке `api/` существуют
3. Проверьте конфигурацию в `vercel.json`

### Если сайт не загружается
1. Проверьте консоль браузера (F12)
2. Убедитесь, что все файлы загружаются
3. Проверьте сетевые запросы в вкладке Network

### Если загрузка файла не работает
1. Проверьте, что файл имеет расширение `.vdf` или `.cfg`
2. Проверьте консоль браузера на ошибки
3. Убедитесь, что API endpoints работают
